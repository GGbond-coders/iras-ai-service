## 1. 系统概述

### 1.1 项目简介

IRAS（Intelligent Resume Analysis System）是一套基于 B/S 架构的**智能简历诊断系统**。系统整合了职位检索、AI 职能画像、智能简历诊断三大核心功能，帮助求职者快速了解自身简历与目标岗位的匹配程度，并获得针对性的改进建议。

### 1.2 核心功能

| 功能模块 | 说明 |
|---------|------|
| **用户认证** | 支持用户名/邮箱登录注册，基于 JWT 的无状态会话管理，Token 有效期 24 小时 |
| **职位检索** | 按职位名称、工作城市、薪资范围进行多字段组合模糊筛选，分页展示结果，支持查看详情 |
| **职能画像** | 输入任意职位名称，AI 自动生成该职位的完整能力画像（硬技能、软技能、工具清单、经验要求、学历要求） |
| **智能诊断** | 上传简历文件（.txt/.pdf/.doc/.docx，最大 10MB），AI 自动分析并与岗位库匹配，生成诊断报告（匹配岗位、匹配分、差距分析、面试建议） |
| **诊断历史** | 查看、回顾和删除过往的诊断记录，支持分页浏览和详情查看 |
| **管理后台** | 管理员专属功能，包含系统概览（统计仪表盘）、用户管理（查看/角色变更/删除）、职位管理（增删改查） |

### 1.3 用户角色

本系统支持两种用户角色：

| 角色 | 说明 |
|------|------|
| **普通用户（user）** | 注册后默认角色，可使用职位检索、职能画像、智能诊断、诊断历史等全部核心功能 |
| **管理员（admin）** | 除普通用户功能外，还可访问管理后台，包括系统概览仪表盘、用户管理、职位管理 |

管理员角色通过数据库 `user` 表的 `role` 字段区分，默认值为 `user`。

## 5. 数据库设计

### 5.1 数据库信息

- **数据库名**：`iras`
- **字符集**：`utf8mb4`
- **排序规则**：`utf8mb4_unicode_ci`

### 5.2 表结构

#### 5.2.1 用户表（`user`）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `bigint` | PRIMARY KEY, AUTO_INCREMENT | 用户唯一标识 |
| `username` | `varchar(50)` | NOT NULL, UNIQUE | 用户名（登录凭据） |
| `password` | `varchar(255)` | NOT NULL | 密码（BCrypt 加密存储） |
| `email` | `varchar(100)` | DEFAULT NULL | 联系邮箱（可选，可用于登录） |
| `role` | `varchar(20)` | NOT NULL, DEFAULT 'user' | 用户角色（user-普通用户, admin-管理员） |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

**索引：**
- `uk_username` — 唯一索引，用户名

#### 5.2.2 职位库表（`job_info`）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `bigint` | PRIMARY KEY, AUTO_INCREMENT | 职位唯一标识 |
| `job_name` | `varchar(100)` | NOT NULL | 职位名称 |
| `company_name` | `varchar(100)` | NOT NULL | 公司名称 |
| `city` | `varchar(50)` | DEFAULT NULL | 工作城市 |
| `salary` | `varchar(50)` | DEFAULT NULL | 薪资范围 |
| `jd_text` | `text` | NOT NULL | 职位描述（JD 内容） |
| `type` | `varchar(50)` | DEFAULT NULL | 职位分类（全职/实习/校招） |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 数据导入时间 |
| `update_time` | `datetime` | ON UPDATE CURRENT_TIMESTAMP | 最后更新时间 |

**索引：**
- `idx_job_name` — 普通索引，职位名称
- `idx_city` — 普通索引，城市
- `idx_fulltext_jd` — 全文索引，职位描述

#### 5.2.3 诊断历史记录表（`diagnosis_record`）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `bigint` | PRIMARY KEY, AUTO_INCREMENT | 记录唯一标识 |
| `user_id` | `bigint` | NOT NULL, FOREIGN KEY → user(id) ON DELETE CASCADE | 关联用户 ID |
| `resume_filename` | `varchar(255)` | NOT NULL | 上传的简历文件名 |
| `resume_content` | `mediumtext` | DEFAULT NULL | 简历提取的文本内容（用于回顾） |
| `diagnosis_result` | `mediumtext` | DEFAULT NULL | 诊断结果 JSON（AI 返回的完整报告） |
| `think_content` | `mediumtext` | DEFAULT NULL | AI 推理过程（think 标签内容） |
| `match_count` | `int` | DEFAULT 0 | 匹配到的岗位数量 |
| `create_time` | `datetime` | DEFAULT CURRENT_TIMESTAMP | 诊断时间 |

**索引：**
- `idx_user_id` — 普通索引，用户 ID

- `idx_create_time` — 普通索引，诊断时间

- `fk_diagnosis_user` — 外键约束，关联 `user(id)`，级联删除

## 6. 后端设计

### 6.1 启动与配置

#### 6.1.1 启动类 `IrasApplication`

- 使用 `@SpringBootApplication` 标记为 Spring Boot 应用
- 使用 `@MapperScan("com.iras.mapper")` 扫描 MyBatis Mapper 接口

#### 6.1.2 应用配置 `application.yml`

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `server.port` | `8080` | 服务端口 |
| `server.servlet.context-path` | `/iras` | 应用上下文路径 |

### 6.2 安全与认证

#### 6.2.1 SecurityConfig — Spring Security 全局配置

**核心配置：**
- **CSRF**：禁用（REST API 无状态，不需要 CSRF 保护）
- **CORS**：允许所有来源（`*`），支持 `GET/POST/PUT/DELETE/OPTIONS`，缓存 1 小时
- **Session**：无状态（`STATELESS`），不使用 HttpSession
- **权限规则**：
  - `/api/auth/**` — 允许匿名访问（登录、注册）
  - `OPTIONS /**` — 允许预检请求
  - 其他所有请求 — 需要认证
- **异常处理**：未认证请求返回 401 JSON 响应 `{"code":401,"message":"未登录或Token已过期"}`
- **过滤器顺序**：`JwtAuthenticationFilter` 在 `UsernamePasswordAuthenticationFilter` 之前执行

**Bean 注册：**
- `PasswordEncoder` — BCrypt 密码编码器
- `AuthenticationManager` — 认证管理器
- `CorsConfigurationSource` — CORS 配置源

**管理员权限控制：** 管理后台接口（`/api/admin/**`）的权限校验在 Controller 层通过 `requireAdmin()` 方法实现，而非在 SecurityConfig 中配置。每个 AdminController 的方法都会先调用 `requireAdmin(authentication)` 检查当前用户是否具有 `admin` 角色，非管理员返回 403 错误。

#### 6.2.2 JwtAuthenticationFilter — JWT 认证过滤器

**处理流程：**

```
请求进入
    │
    ▼
从 Header 提取 Token
(格式: "Bearer <token>")
    │
    ▼
Token 有效？
    ├─ 是 → 解析用户名 → 设置 SecurityContext 认证信息
    └─ 否 → 跳过认证（后续由 SecurityConfig 决定是否放行）
    │
    ▼
继续过滤器链
```

**关键方法：**
- `doFilterInternal()` — 每次请求执行的主过滤逻辑
- `extractToken()` — 从 `Authorization` 请求头提取 Bearer Token

#### 6.2.3 JwtUtil — JWT 工具类

**功能：**

| 方法 | 说明 |
|------|------|
| `generateToken(username)` | 生成 JWT Token，包含 subject（用户名）、签发时间、过期时间 |

| `validateToken(token)` | 验证 Token 是否有效（签名正确 + 未过期） |

**签名算法：** HMAC-SHA512（HS512），密钥从配置文件读取

### 6.4 Service 层

#### 6.4.1 AuthService / AuthServiceImpl — 认证服务

**注册流程：**

```
接收注册请求
    │
    ▼
用户名已存在？ ──是──→ 返回错误 "用户名已存在"
    │否
    ▼
邮箱已被注册？ ──是──→ 返回错误 "邮箱已被注册"
    │否
    ▼
BCrypt 加密密码
    │
    ▼
插入数据库
    │
    ▼
生成 JWT Token
    │
    ▼
返回 Token + 用户信息
```

**登录流程：**

```
接收登录请求
    │
    ▼
按用户名查找用户
    │
未找到 → 按邮箱查找用户
    │
仍未找到 → 返回错误 "用户名或密码错误"
    │
    ▼
BCrypt 验证密码
    │
不匹配 → 返回错误 "用户名或密码错误"
    │
    ▼
生成 JWT Token
    │
    ▼
返回 Token + 用户信息
```

**特点：**
- 支持用户名或邮箱两种方式登录
- 密码使用 BCrypt 加密存储
- 登录成功/注册成功均返回 JWT Token

#### 6.4.2 JobService / JobServiceImpl — 职位服务

**搜索逻辑：**
1. 计算分页偏移量 `offset = (page - 1) * size`
2. 调用 Mapper 执行多条件模糊搜索
3. 调用 Mapper 统计匹配总数
4. 封装为 `PageResult` 返回（含 records、total、page、size、totalPages）

**默认分页参数：** page=1, size=20

#### 6.4.3 DifyService / DifyServiceImpl — AI 服务

**职能画像调用流程：**

```
接收职位名称
    │
    ▼
构建 Workflow 请求体
(inputs: { job_name: "..." })
    │
    ▼
POST https://api.dify.ai/v1/workflows/run
(Header: Bearer <job-profile-key>)
    │
    ▼
解析响应 → 返回 result 字段
```

**简历诊断调用流程：**

```
接收简历文件
    │
    ▼
第一步：上传文件到 Dify
POST /v1/files/upload
(multipart/form-data: file + user)
    │
    ▼
获取 upload_file_id
    │
    ▼
第二步：调用 Workflow
POST /v1/workflows/run
(inputs: { resume_text: [{transfer_method, upload_file_id, type}] })
    │
    ▼
解析响应 → 返回 result 字段
```

**技术细节：**
- 使用 `HttpURLConnection` 直接发起 HTTP 请求（未使用 RestTemplate/WebClient）
- 连接超时：文件上传 60 秒，Workflow 调用 300 秒（5 分钟）
- 文件上传使用手动拼接 multipart/form-data 请求体
- 请求模式：`streaming`（SSE 流式响应），通过 `parseSseStream()` 解析 `text_chunk` 和 `workflow_finished` 事件
- 内置重试机制：遇到 502/503/504 暂时性错误时自动重试最多 2 次，间隔 5 秒

### 7.2 路由设计


**路由守卫逻辑：**

- 目标路由 `adminOnly === true` 且用户非管理员（`isAdmin` 为 false）→ 重定向到 `/`
- 其他情况 → 正常放行

**页面缓存：** 使用 `<keep-alive>` 缓存已访问的路由组件，避免重复渲染。数据刷新通过 `onActivated` 钩子实现（替代 `onMounted`），确保每次从缓存恢复时重新拉取最新数据。

## 9. AI 引擎集成

### 9.1 Dify Workflow API

本系统通过 Dify 平台的 Workflow API 实现两个 AI 功能：

| 功能 | API Key | 输入 | 输出 |
|------|---------|------|------|
| 职能画像 | `dify.job-profile-key` | 职位名称（文本） | 能力画像 JSON |
| 简历诊断 | `dify.resume-diagnosis-key` | 简历文件（文档） | 匹配报告 JSON/文本 |

### 9.2 API 调用规范

**基础地址：** `https://api.dify.ai/v1`

**认证方式：** `Authorization: Bearer <api-key>`

**请求格式：**

```json
{
  "inputs": {
    "<key>": "<value>"  // 或文件引用数组
  },
  "response_mode": "streaming",
  "user": "iras-user"
}
```

**SSE 流式响应解析：**

Dify streaming 模式返回 Server-Sent Events 格式，关键事件：

优先从 `workflow_finished` 的 `outputs.result` 取结果，若无则返回拼接的 `text_chunk` 内容。

**文件引用格式（简历诊断）：**


**响应解析：**
- 成功响应：`data.outputs.result` 字段包含 AI 生成的结果

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .csrf(AbstractHttpConfigurer::disable)    // 禁用 CSRF。原因：REST API 使用 JWT 无状态认证，
                                                   // 不依赖 Cookie，无需 CSRF 保护

        .cors(cors -> cors.configurationSource(corsConfigurationSource()))
                                                   // 启用 CORS，使用自定义配置源

        .sessionManagement(session ->
            session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
                                                   // 无状态会话：不创建 HttpSession，
                                                   // 每次请求都通过 JWT 重新认证

        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/**").permitAll()
                                                   // 放行 /api/auth/* 下所有接口（登录、注册）
            .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()
                                                   // 放行所有 OPTIONS 预检请求（CORS 规范要求）
            .anyRequest().authenticated()          // 其余所有接口必须经过认证
        )

        .exceptionHandling(ex -> ex
            .authenticationEntryPoint((request, response, authException) -> {
                // 自定义 401 响应：返回 JSON 而非默认的 HTML 登录页
                response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
                response.setContentType(MediaType.APPLICATION_JSON_VALUE);
                response.setCharacterEncoding("UTF-8");
                objectMapper.writeValue(response.getWriter(), Result.error(401, "未登录或Token已过期"));
            })
        )

        .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
        // 将 JWT 过滤器插入到 UsernamePasswordAuthenticationFilter 之前
        // 确保在 Spring Security 默认认证流程之前完成 JWT 解析

    return http.build();
}
```

#### 11.1.3 JwtAuthenticationFilter.java — JWT 认证过滤器

**文件路径：** `backend/src/main/java/com/iras/config/JwtAuthenticationFilter.java`

**职责：** 拦截每个 HTTP 请求，提取并验证 JWT Token，设置认证信息到 SecurityContext。

**执行流程图：**

```
HTTP 请求进入
    │
    ▼
doFilterInternal() 被调用
    │
    ▼
extractToken(request) ─── 从 Header 提取 Token
    │                      格式: "Authorization: Bearer eyJ..."
    │                      substring(7) 去掉 "Bearer " 前缀
    ▼
Token 非空且有效？
    ├─ 是 → jwtUtil.getUsernameFromToken(token) 解析用户名
    │       创建 UsernamePasswordAuthenticationToken
    │       (principal=username, credentials=null, authorities=[])
    │       存入 SecurityContextHolder.getContext().setAuthentication(...)
    │       └─ 后续可通过 SecurityContextHolder 获取当前登录用户
    └─ 否 → 跳过认证设置（SecurityContext 中无认证信息）
    │
    ▼
filterChain.doFilter(request, response) → 继续后续过滤器链
```

**关键代码解析：**

```java
@Override
protected void doFilterInternal(HttpServletRequest request,
                                HttpServletResponse response,
                                FilterChain filterChain) throws ServletException, IOException {
    // 第一步：从请求头提取 JWT Token
    String token = extractToken(request);

    // 第二步：验证 Token 有效性（签名正确 + 未过期）
    if (StringUtils.hasText(token) && jwtUtil.validateToken(token)) {
        // 第三步：从 Token 中解析用户名（即 Token 的 subject 字段）
        String username = jwtUtil.getUsernameFromToken(token);

        // 第四步：创建 Spring Security 认证令牌
        // 参数说明：
        //   - principal (username): 当前认证用户的身份标识
        //   - credentials (null): 不需要密码（已通过 JWT 认证）
        //   - authorities ([]): 暂未设置角色权限
        UsernamePasswordAuthenticationToken authentication =
                new UsernamePasswordAuthenticationToken(username, null, Collections.emptyList());

        // 第五步：将认证信息存入 SecurityContext
        // 后续的 Controller 中可通过 @AuthenticationPrincipal 或
        // SecurityContextHolder.getContext().getAuthentication() 获取当前用户
        SecurityContextHolder.getContext().setAuthentication(authentication);
    }

    // 无论认证是否成功，都继续执行后续过滤器
    // 如果接口需要认证但 Token 无效，SecurityConfig 中的权限规则会拒绝访问
    filterChain.doFilter(request, response);
}

private String extractToken(HttpServletRequest request) {
    String bearerToken = request.getHeader("Authorization");
    // 检查格式: "Bearer <token>"
    if (StringUtils.hasText(bearerToken) && bearerToken.startsWith("Bearer ")) {
        return bearerToken.substring(7);  // 截取 "Bearer " 之后的部分
    }
    return null;  // 无 Token 或格式不正确
}
```

#### 11.1.4 JwtUtil.java — JWT 工具类

**文件路径：** `backend/src/main/java/com/iras/util/JwtUtil.java`

**职责：** 封装 JWT 的生成、解析、验证逻辑。

**Token 结构：**

```
eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJ0ZXN0dXNlciIsImlhdCI6MTcxMzAwMDAwMCwiZXhwIjoxNzEzMDg2NDAwfQ.xxxxx
│←──────── Header ────────→│←──────────────── Payload ──────────────────────→│←─ Signature ─→│

Header:  {"alg":"HS512"}                           → 算法声明
Payload: {"sub":"testuser","iat":1713000000,...}    → 用户名、签发时间、过期时间
Signature: HMAC-SHA512(Header + "." + Payload, secret) → 签名
```

**三个核心方法解析：**

```java
// ============ 生成 Token ============
public String generateToken(String username) {
    return Jwts.builder()
        .subject(username)                                          // subject = 用户名
        .issuedAt(new Date())                                       // iat = 当前时间
        .expiration(new Date(System.currentTimeMillis() + expiration)) // exp = 当前时间 + 24小时
        .signWith(getSigningKey())                                  // 使用 HS512 签名
        .compact();                                                  // 生成最终字符串
    // 返回: "eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJ0ZXN0dXNlciIs..."
}

// ============ 解析 Token ============
public String getUsernameFromToken(String token) {
    return Jwts.parser()
        .verifyWith(getSigningKey())      // 设置签名验证密钥
        .build()
        .parseSignedClaims(token)         // 验证签名 + 解析 Payload
        .getPayload()
        .getSubject();                    // 提取 subject 字段 = 用户名
    // 如果签名不正确或 Token 过期，会抛出 JwtException
}

// ============ 验证 Token ============
public boolean validateToken(String token) {
    try {
        Jwts.parser()
            .verifyWith(getSigningKey())
            .build()
            .parseSignedClaims(token);    // 尝试解析
        return true;                       // 成功 → Token 有效
    } catch (JwtException | IllegalArgumentException e) {
        return false;                      // 失败 → Token 无效（过期/篡改/格式错误）
    }
}

// ============ 密钥生成 ============
private SecretKey getSigningKey() {
    return Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
    // 将配置文件中的密钥字符串转为 SecretKey 对象
    // 密钥长度需 ≥ 64 字节（512 位）以满足 HS512 要求
}
```

```java
public Result<Map<String, Object>> register(RegisterRequest request) {
    // 第一步：用户名唯一性检查
    if (userMapper.findByUsername(request.getUsername()) != null) {
        return Result.error(400, "用户名已存在");
    }

    // 第二步：邮箱唯一性检查（仅在提供了邮箱时检查）
    if (request.getEmail() != null && userMapper.findByEmail(request.getEmail()) != null) {

    }

    // 第三步：构建用户实体
    User user = new User();
    user.setUsername(request.getUsername());
    user.setPassword(passwordEncoder.encode(request.getPassword()));
    // ↑ BCrypt 加密：每次加密结果不同（内置随机盐），但 matches() 可正确比较
    user.setEmail(request.getEmail());

    // 第四步：插入数据库（MyBatis 自动生成的 INSERT 语句）
    userMapper.insert(user);
    // @Options(useGeneratedKeys = true, keyProperty = "id") 会将自增 ID 回填到 user.id

    // 第五步：生成 JWT Token
    String token = jwtUtil.generateToken(user.getUsername());

    Map<String, Object> data = new HashMap<>();
    data.put("token", token);         // 前端后续请求都携带此 Token
    data.put("username", user.getUsername());
    data.put("email", user.getEmail());

    return Result.success("注册成功", data);
}
```

```java
@Override
public Result<Map<String, Object>> login(LoginRequest request) {
    // 第一步：支持用户名或邮箱登录
    User user = userMapper.findByUsername(request.getUsername());
    if (user == null) {
        user = userMapper.findByEmail(request.getUsername());
        // ↑ 如果用户名查找失败，尝试用同一字段作为邮箱查找
        // 前端登录表单的 "用户名或邮箱" 字段复用
    }

    if (user == null) {
        return Result.error(401, "用户名或密码错误");
        // ↑ 安全考虑：不区分 "用户不存在" 和 "密码错误"，防止枚举攻击
    }

    // 第二步：BCrypt 密码验证
    if (!passwordEncoder.matches(request.getPassword(), user.getPassword())) {
        // passwordEncoder.matches(明文密码, 数据库中的BCrypt哈希)
        // 内部原理：从哈希中提取盐值，用盐值加密明文，比较结果
        return Result.error(401, "用户名或密码错误");
    }

    // 第三步：生成 JWT Token（登录成功）
    String token = jwtUtil.generateToken(user.getUsername());

    Map<String, Object> data = new HashMap<>();
    data.put("token", token);
    data.put("username", user.getUsername());
    data.put("email", user.getEmail());

    return Result.success("登录成功", data);
}
```

```java
@Override
public PageResult<JobInfo> searchJobs(JobSearchRequest request) {
    // 第一步：处理分页参数默认值
    int page = request.getPage() != null ? request.getPage() : 1;    // 默认第 1 页
    int size = request.getSize() != null ? request.getSize() : 20;   // 默认每页 20 条

    // 第二步：计算 SQL 分页偏移量
    int offset = (page - 1) * size;
    // 例如 page=2, size=20 → offset=20（跳过前 20 条）

    // 第三步：查询当前页数据
    List<JobInfo> records = jobMapper.searchJobs(
            request.getJobName(),    // 职位名称（可为 null）
            request.getCity(),       // 城市（可为 null）
            request.getSalaryMin(),  // 最低薪资（可为 null）
            request.getSalaryMax(),  // 最高薪资（可为 null）
            offset, size);

    // 第四步：查询总记录数（同样的搜索条件）
    long total = jobMapper.countSearchJobs(
            request.getJobName(), request.getCity(),
            request.getSalaryMin(), request.getSalaryMax());

    // 第五步：组装分页结果
    PageResult<JobInfo> result = new PageResult<>();
    result.setRecords(records);                                      // 当前页数据
    result.setTotal(total);                                          // 总记录数
    result.setPage(page);                                            // 当前页码
    result.setSize(size);                                            // 每页大小
    result.setTotalPages((int) Math.ceil((double) total / size));    // 总页数（向上取整）

    return result;
}
```

```java
private String uploadFile(MultipartFile file) {
    // 使用手动拼接 multipart/form-data 请求体（未使用第三方 HTTP 库）
    String boundary = "----WebKitFormBoundary" + System.currentTimeMillis();
    // ↑ multipart 边界字符串，用于分隔不同字段

    HttpURLConnection connection = (HttpURLConnection) new URL(uploadUrl).openConnection();
    connection.setRequestMethod("POST");
    connection.setRequestProperty("Authorization", "Bearer " + resumeDiagnosisKey);
    connection.setRequestProperty("Content-Type", "multipart/form-data; boundary=" + boundary);
    connection.setDoOutput(true);     // POST 请求需要设置为 true
    connection.setConnectTimeout(60000);  // 连接超时 60 秒
    connection.setReadTimeout(60000);     // 读取超时 60 秒

    // 手动拼接 multipart 请求体
    OutputStream os = connection.getOutputStream();

    // 字段 1：file（文件内容）
    os.write(("--" + boundary + "\r\n").getBytes());                          // 字段开始边界
    os.write(("Content-Disposition: form-data; name=\"file\"; filename=\"" +
              file.getOriginalFilename() + "\"\r\n").getBytes());             // 字段元信息
    os.write(("Content-Type: " + file.getContentType() + "\r\n\r\n").getBytes()); // 文件 MIME 类型
    os.write(file.getBytes());                                                // 文件二进制内容
    os.write("\r\n".getBytes());                                              // 字段结束换行

    // 字段 2：user（用户标识）
    os.write(("--" + boundary + "\r\n").getBytes());
    os.write("Content-Disposition: form-data; name=\"user\"\r\n\r\n".getBytes());
    os.write("iras-user\r\n".getBytes());

    // 结束边界
    os.write(("--" + boundary + "--\r\n").getBytes());

    // 读取响应，解析 JSON 提取文件 ID
    JsonNode root = objectMapper.readTree(responseBody);
    return root.get("id").asText();  // 返回 Dify 平台分配的 upload_file_id
}
```

```java
    HttpURLConnection connection = (HttpURLConnection) new URL(urlStr).openConnection();
    connection.setRequestMethod("POST");
    connection.setRequestProperty("Authorization", "Bearer " + apiKey);
    connection.setRequestProperty("Content-Type", "application/json");
    connection.setDoOutput(true);
    connection.setConnectTimeout(300000);  // 5 分钟超时（AI 推理耗时较长）
    connection.setReadTimeout(300000);

    // 写入请求体
    OutputStream os = connection.getOutputStream();
    os.write(requestBody.getBytes(StandardCharsets.UTF_8));

    // 读取响应
    int responseCode = connection.getResponseCode();
    InputStream inputStream = (responseCode >= 200 && responseCode < 300)
            ? connection.getInputStream()     // 成功响应
            : connection.getErrorStream();    // 错误响应
    String responseBody = new String(inputStream.readAllBytes(), StandardCharsets.UTF_8);

    if (responseCode >= 200 && responseCode < 300) {
        // 解析 Dify 响应结构：{ "data": { "outputs": { "result": "AI生成的内容" } } }
        JsonNode root = objectMapper.readTree(responseBody);
        JsonNode outputs = root.path("data").path("outputs");
        if (outputs.has("result")) {
            return outputs.get("result").asText();  // 提取 result 字段
        }
        return responseBody;  // 无 result 字段时返回完整响应
    }
}
```

#### 11.1.11 DifyController 诊断记录自动保存

**文件路径：** `backend/src/main/java/com/iras/controller/DifyController.java`

**新增功能：** 诊断成功后自动保存记录到 `diagnosis_record` 表。

在 `diagnoseResume()` 方法中，调用 Dify API 成功后，会执行 `saveDiagnosisRecord()`：

```java
@PostMapping("/diagnose")
public Result<String> diagnoseResume(@RequestParam("file") MultipartFile file,
                                     Authentication authentication) {
    // ... 调用 Dify API ...
    String result = difyService.diagnoseResume(file);

    // 诊断成功后保存记录
    try {
        saveDiagnosisRecord(file.getOriginalFilename(), result, authentication);
    } catch (Exception e) {
        log.warn("保存诊断记录失败", e);  // 保存失败不影响诊断结果返回
    }

    return Result.success(result);
}
```

**saveDiagnosisRecord() 方法逻辑：**
1. 从 `Authentication` 获取当前用户
2. 从 AI 结果中提取 `<think>` 标签内容（推理过程）→ 存入 `thinkContent`
3. 从结果中移除 `<think>` 标签，得到干净的诊断结果 → 存入 `diagnosisResult`
4. 尝试解析 JSON 计算匹配岗位数
5. 构建 `DiagnosisRecord` 并调用 `diagnosisService.saveRecord()` 持久化

> **注意**：`diagnosisResult` 存储的是去除 think 标签后的干净结果（`cleanResult`），推理过程单独存储在 `thinkContent` 字段，确保前端展示时两者正确分离。

```javascript
export const useUserStore = defineStore('user', () => {
  // ===== 状态定义 =====
  // 从 localStorage 初始化（页面刷新时恢复登录状态）
  const token = ref(localStorage.getItem('token') || '')
  const username = ref(localStorage.getItem('username') || '')
  const email = ref(localStorage.getItem('email') || '')

  // ===== 操作方法 =====
  function setUser(userData) {
    // 更新内存状态（响应式，自动触发视图更新）
    token.value = userData.token
    username.value = userData.username
    email.value = userData.email || ''
    // 持久化到 localStorage（页面刷新后可恢复）
    localStorage.setItem('token', userData.token)
    localStorage.setItem('username', userData.username)
    localStorage.setItem('email', userData.email || '')
  }

  function logout() {
    // 清除内存状态
    token.value = ''
    username.value = ''
    email.value = ''
    // 清除 localStorage
    localStorage.removeItem('token')
    localStorage.removeItem('username')
    localStorage.removeItem('email')
  }

  return { token, username, email, setUser, logout }
})
```

```
登录成功 → setUser() → 同时写入 ref + localStorage
页面刷新 → Store 初始化 → 从 localStorage 读取 → 恢复登录状态
退出登录 → logout() → 同时清空 ref + localStorage
```

#### 11.2.9 Diagnosis.vue — 智能诊断页面

**文件路径：** `frontend/src/views/Diagnosis.vue`

**核心逻辑 — 诊断结果解析：**

```javascript
async function diagnose() {
  loading.value = true

  const res = await difyApi.diagnoseResume(selectedFile.value)
  // 发送: POST /iras/api/dify/diagnose (multipart/form-data)
  // 响应: { code: 200, data: "AI返回的内容" }

  let data = res.data

  // ===== 第一步：提取 <think> 标签（AI 推理过程） =====
  const thinkMatch = data.match(/<think>([\s\S]*?)<\/think>/)
  // [\s\S]*? 匹配任意字符（包括换行），非贪婪模式
  if (thinkContent) {
    thinkContent.value = thinkMatch[1].trim()  // 提取标签内容
    data = data.replace(/<think>[\s\S]*?<\/think>/, '').trim()  // 从数据中移除标签
  }

  // ===== 第二步：尝试解析 JSON =====
  try {
    const parsed = JSON.parse(data)

    if (Array.isArray(parsed)) {
      // 情况 A：直接是数组 → 匹配结果列表
      // [{ matched_job, matched_score, matched_reason, gap_points, interview_advice }, ...]
      matches.value = parsed

    } else if (parsed.matches) {
      // 情况 B：对象中有 matches 字段
      matches.value = parsed.matches

    } else if (parsed.result) {
      // 情况 C：对象中有 result 字段（可能是嵌套的 JSON 字符串）
      const inner = parsed.result
      // 递归提取 <think> 标签
      const innerThink = inner.match(/<think>([\s\S]*?)<\/think>/)
      if (innerThink) thinkContent.value = innerThink[1].trim()

      const cleanJson = inner.replace(/<think>[\s\S]*?<\/think>/, '').trim()
      try {
        const innerParsed = JSON.parse(cleanJson)
        matches.value = Array.isArray(innerParsed) ? innerParsed : [innerParsed]
      } catch {
        rawResult.value = cleanJson  // 解析失败则作为纯文本处理
      }
    } else {
      matches.value = [parsed]  // 单个对象包装为数组
    }
  } catch {
    rawResult.value = data  // JSON 解析失败则作为纯文本/Markdown 处理
  }
}
```

### 12.2 安全机制汇总

| 机制 | 实现 |
|------|------|
| 密码加密 | BCrypt（Spring Security PasswordEncoder） |
| 认证方式 | JWT Bearer Token（无状态） |
| Token 有效期 | 24 小时 |
| CSRF | 禁用（REST API + 无状态 Session） |
| CORS | 允许所有来源（开发环境） |
| 接口权限 | `/api/auth/**` 公开，其余需认证 |
| 文件上传限制 | 10MB |
| SQL 注入防护 | MyBatis 参数化查询（`#{}` 占位符） |
| 管理员权限控制 | Controller 层 `requireAdmin()` 方法校验角色 |
| 诊断记录归属校验 | 查询/删除操作校验 userId 匹配，防止越权 |

### 12.3 错误码对照表

|------------|--------|------|
